# AGENTS.md

## 项目最高规则

- 每次任务开始先读取 `AGENTS.md`、`MEMORY.md`、`对话.md`，存在 `HANDOFF.md` 时也要读取。
- 结束前必须把用户原始需求、处理过程、验证步骤和遗留问题写入 `MEMORY.md` 与 `对话.md`。
- 不硬编码 API Key、Token、Cookie 或任何真实密钥；前端绝不直连带密钥模型 API。
- 面向未成年人功能必须默认数据最小化、监护人同意、AI 生成标识、可清除本地数据。
- UI 必须保持日记手账风、暖白纸张背景、红色边距线、液态玻璃、果冻动效，强制浅色体验。
- 不做公开社区、陌生人互动、排行榜、付费诱导或真实儿童长期档案。
- 共读材料只能来自自建、公版、授权或用户上传内容。
- 作业/练字点评不得显示百分比、排名、羞辱性分数；反馈顺序是先鼓励、再观察、再给一个小改进。
- 所有云端模型能力必须有 Mock/本地降级，保证黑客松演示不中断。
- 删除文件前确认不是 `references/` 或关键上下文；禁止使用 `git reset --hard` 等破坏性命令，除非用户明确要求。

## 跳跳学长 Demo 验收底线

- 主链路可演示：监护人同意 -> 首页 -> 问答 -> 共读 -> 拍照点评 -> 生图/视频 -> 小游戏 -> 家长卡。
- 服务端代理路径包括 `/api/chat`、`/api/tts`、`/api/vision`、`/api/image`、`/api/video/jobs`。
- 真实 API 优先，失败后降级并在界面诚实标识。
- `npm run build`、`npm run test` 和密钥扫描必须在收尾时运行并记录结果。

## UI 风格沉淀

- 可参考 Mastercard 式暖奶油底、圆形轨道、胶囊导航、黑色主 CTA 和克制留白，但不得复制 Mastercard 标志、红黄双圆、官方字体或卡面资产。
- 可参考 Duolingo 式角色陪伴、轻量游戏化和明亮点缀色，但必须使用原创角色，不复制绿色猫头鹰、官方插画、角色轮廓或品牌语气。
- 界面禁止使用 emoji；可爱感通过原创像素小人、CSS 图形、lucide 图标、纸张细节和果冻动效表达。
- 游戏化只做个人陪伴和亲子任务，不做排行榜、公开比较、羞辱性分数或百分比排名。
